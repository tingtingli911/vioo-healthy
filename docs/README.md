## 文档更新规范

### 何时更新文档
1. **新增功能**：添加新功能模块时，必须更新 `03-功能模块.md` 和 `CHANGELOG.md`
2. **API 变更**：修改或新增 API 接口时，必须更新 `04-API文档.md`
3. **数据库变更**：修改数据模型时，必须更新 `05-数据库设计.md`
4. **架构调整**：技术栈或架构变更时，必须更新 `02-技术架构.md`
5. **规范调整**：开发规范变更时，必须更新 `06-开发规范.md`

### 文档更新流程
1. 开发功能前，先查看相关文档
2. 开发过程中，同步更新文档
3. 代码提交前，确保文档已更新
4. 在 `CHANGELOG.md` 中记录变更
